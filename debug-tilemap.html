Phaser Tilemap Positioning Analysis

The Problem:

When you create a Phaser tilemap with:

const map = scene.make.tilemap({
    tileWidth: 48,
    tileHeight: 48,
    width: 37,
    height: 37
});
const tilemapLayer = map.createBlankLayer('layer', tileset, worldX, worldY);

Expected Behavior:

  • Chunk (0,0) at position (0, 0) - covers pixels [0, 1776) x [0, 1776)
  • Chunk (1,0) at position (1776, 0) - covers pixels [1776, 3552) x [0, 1776)
  • Chunk (0,1) at position (0, 1776) - covers pixels [0, 1776) x [1776, 3552)

Actual Behavior:

THE TILEMAP DIMENSIONS ARE BEING CREATED AS 37x37 TILES

But each chunk should render exactly 1776x1776 pixels (37 tiles * 48px = 1776px)

Hypothesis:

The tilemaps are being created correctly, but:

  1. Maybe the tilemap size calculation is wrong?
  2. Maybe the positioning worldX, worldY is being interpreted incorrectly?
  3. Maybe there's an issue with how tiles are being placed?

Key Code Location:

BiomeChunkSystem.js, line 301-306 (tilemap creation)

BiomeChunkSystem.js, line 377 (tilemap layer creation with worldX, worldY)

Solution:

Need to verify if tilemap is actually 1776x1776 pixels or if it's being created at a different size!
